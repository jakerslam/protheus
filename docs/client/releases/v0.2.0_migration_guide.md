# v0.2.0 Migration Guide (Draft)

## Audience

Operators upgrading from pre-`v0.2.0` branches to the conduit-first security posture.

## Key Changes

- Conduit-first TS -> Rust lane execution hardening across additional runtime lanes.
- Security credibility baseline added:
  - CodeQL workflow
  - Dependabot updates
  - release SBOM workflow
  - security posture + disclosure/bounty docs

## Upgrade Steps

1. Pull latest `main` and install dependencies:
   - `npm ci`
   - `cargo build -p infring-ops-core --bin infring-ops`
2. Validate invariants and hardening:
   - `NODE_PATH=$PWD/node_modules npm run -s formal:invariants:run`
   - `cargo run --manifest-path core/layer0/ops/Cargo.toml --bin infring-ops -- enterprise-hardening run --strict=1`
3. Re-run benchmark matrix:
   - `target/debug/infring-ops benchmark-matrix run --refresh-runtime=1`

## Verification

- No unmerged files in worktree.
- `enterprise-hardening` gate passes.
- Formal invariants report `ok: true`.
- Benchmark report includes measured `cold_start_ms`, `idle_memory_mb`, and full `install_size_mb`.
