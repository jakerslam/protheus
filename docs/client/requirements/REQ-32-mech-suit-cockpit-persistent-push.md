# REQ-32 — Mech-Suit Cockpit Runtime (Persistent + Push + Ambient Memory)

Status: in_progress  
Owner: Runtime + Architecture  
Updated: 2026-03-08

## Objective
Deliver mech-suit behavior without violating Layer 0 invariants by making cockpit runtime:
- persistent at daemon/cognition layer,
- push-fed via conduit attention streams,
- and ambient-memory aware with bounded identity hydration.

## Safety Boundaries (Non-Negotiable)
- Layer 0 remains deterministic source-of-truth.
- No mutable cognition state is moved into Layer 0 correctness authority.
- Shell/core transport remains conduit + scrambler only.

## Functional Requirements

### REQ-32-001 Persistent attach-or-start cockpit entrypoint
- `infringd attach` must provide daemon attach semantics.
- If daemon is not running and autostart is enabled, attach must start daemon first.
- Attach response must include deterministic receipt + runtime status envelope.

### REQ-32-002 Conduit-backed subscription lane
- `infringd subscribe` must consume attention queue updates through conduit-backed lane execution.
- Stream output must be structured JSON envelopes with bounded batch size and cadence.
- Consumer ID and cursor progression must remain deterministic.

### REQ-32-003 Resident ambient memory status surface
- Daemon must maintain a resident memory snapshot artifact under runtime state.
- Snapshot must include memory ambient health, sqlite/cached paths, and hotset indicators.
- Status/attach responses must include resident memory summary.

### REQ-32-004 Incremental identity hydration budget
- Startup identity loading must be policy-driven and token-budgeted.
- Base files load first under a strict startup budget; deferred pages are marked lazy.
- Hydration result is persisted as a deterministic state artifact.

## Acceptance Criteria
1. `node client/runtime/systems/ops/infringd.ts attach` returns `type=infring_daemon_attach` with status payload.
2. `node client/runtime/systems/ops/infringd.ts subscribe --once=1` returns a start envelope and at least one batch envelope.
3. `client/runtime/local/state/ops/resident_memory/latest.json` is generated by runtime heartbeat.
4. `client/runtime/local/state/ops/identity_hydration/latest.json` is generated with budget/load/deferred fields.
5. No Layer 0 invariant/receipt boundary is weakened.

## Implementation Notes
- Initial subscription lane can be polling-backed at daemon boundary while remaining push-style for cockpit consumers.
- Follow-on work should add long-lived native conduit stream transport for lower latency.
